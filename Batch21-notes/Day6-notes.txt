# Topics:

1. Pull Request
2. Merging

In my project, we are maintaining pull request, the purpose of pull request is every time when we wanted to merge the code/changes from my branch to master/main branch, we need to take the approval from the approver/reviewer.

Once my reviewer approved my PR, then we can merge the changes from my branch to master branch.

this is the strategy we are maintaining in my current organisation.


# Merging: Combining the changes / Integrate the changes from one branch to another branch

# Merge conflict:

Merge conflict will come when we do not have the up to date code when we are trying to combining the changes from my branch to master branch


# How do you resolve the merge conflict??


I need to compare my code with the master branch. if everything is up to date, then I can merge.

If I do not have the up to date code, then we need to take the changes from the master which I do not have, then I can merge the changes to my branch.

Once everything is good, then I go with combining the changes to master branch.

This is how we are doing in the current organization.
